事件处理器

事件处理器

不需要传递参数，可以不写（），会自动传递一个event对象

需要传递参数的时候要写（），且不会自动传递event对象，但是可以手动添加事件对象，实参有固定写法$event

可以直接写代码,页面中不需要加this，但是只能写简单表达式，而且不太希望在页面解构中写太多逻辑减少可读性所以不推荐

事件修饰符

阻止事件冒泡

修饰符

.stop:阻止冒泡，在模态中应用

.prevent:阻止默认行为

.capture:在捕获阶段触发方法

.self:只有点击事件的事件源是自己的时候才会触发，所以不会阻止事件冒泡，但是不会触发点击事件

.once:让方法只能执行一次

.passive:将内核线程查询跳过，每次事件产生以后，浏览器都会去查询是否有preventDefault阻止该次事件的默认动作，我们加上.passive就是为了告诉浏览器不用查询了，我们没有使用preventDefalut阻止默认动作，一般用在滚动监听，@scroll,@toucnmove中，移动每个像素都会产生一次事件，使用.pressive可以大大提升滑动的流畅性。

.enter:按键修饰符，只有按下特定的值才会执行方法,也有上下左右，空格shift，control等，也可以使用.键值来定义

也可以两个按键修饰符一起使用，必须两个键同时按才能触发

否则需要在函数体中使用keycode来定位按键

  2. v-show动态显示和隐藏

  3. v-if条件渲染，动态创建和移除,template是一个包装元素，不会真正地创建到页面上，所以不会影响dom元素但是可以根据这个包装元素上写逻辑

  4. v-bind动态绑定属性，可以简写为：

  5. v-for遍历数组的指令，v-for也可以遍历对象，而且有三个参数（对象属性值，对象属性名，对象属性的索引），但是一般很少遍历对象 特殊：v-for(n in 10）输出的是1到10的遍历 v-for必须要有key值，key可以用来跟踪每一个节点的身份，方便虚拟dom的比较。如果key用索引值每次改变是根据虚拟dom节点比较来进行改变而不是修改正确对应的dom节点，比如有1，2，3个节点，删除2的操作是把2改成3，然后删掉3。index只能适用于在数组的最后添加元素的list或者以后不会改变的list。重用和重新排序现有的元素，理想的key值是每项都有且唯一的id

Vue基础

Vue安装方式

第一种是创建一个.html文件，然后使用script标签进行引入

第二种是直接使用vue-cli模块化的方式。

Vue原理

el是element的缩写，初始化el中的标签作为vue的地盘。

在el初始化了以后的地方，{{ }}就是模板语法-表达式（三目运算符也算是表达式）

通过data属性可以把状态挂载到vue实例上，状态改变页面也会自动更新

在vue中有方法监听到了状态被修改了，在这个方法中把dom也给修改了，但是这整个的过程是在vue内部执行的。所以Vue的原理是通过使用Object.defineProperty这个方法中的get方法和set方法的拦截去实现的。

每个组件实例都对应一个watcher实例，它会在组件渲染的过程中把接触过的数据property记录为依赖，把节点和状态进行一一的关联，之后当依赖项的setter触发时，它会通知watcher，从而使他关联的组件重新渲染。

Vue会根据新老dom生成新老虚拟dom树进行比较找到最优解，然后去更新真实dom，从而尽量减少dom操作

mvvm架构模式

model：数据层

view：视图层

viewmodel：

view层数据改变—>通知viewmodel—>model数据改变

model数据改变—>viewmodel做了getset拦截，自动得到通知，自动通知view层修改—>view层修改数据

解决了mvp大量的手动view和model同步的问题，提供了双向绑定机制，提高了代码的可维护性，对于大型的图形应用程序，视图状态较多，viewmodel的构建和维护的成本可能会比较高。

class

class可以理解为语法糖，将js只能通过构造函数创建实例的方法进行了补充

class的特性

  1. class的数据类型是一个函数

  2. class的原型的constructor指向class

  3. 通过new关键字创建的实例的constructor指向class，类似于原型链

  4. class内部的方式实际上都是定义在类的prototype上

  5. 通过类创建的对象的本质是调用类的constructor，如果类没有定义constructor，那么会在使用的时候默认添加

  6. class不能直接调用，需要使用new关键字，这是class虽然是函数，但是和函数不同的地方

  7. class内部的方法指向的是实例，class内部是严格模式，没有调用者的话函数内部的this就是undefined，在严格模式下也是不存在变量提升的，使用箭头函数的话就可以让class内部的方法中的this指向class实例，但是同时class里面的属性就会跑到实例上而非prototype上。

  8. getter和setter

  9. 动态设置属性名

  10. 通过在属性和方法前面添加关键字static，这些属性和方法就不会被实例继承，只能通过【类名.属性名/方法名】的方式来获取,静态方法指向的是类而不是类的实例，同时静态方法是允许重复命名的。

class的继承

class的继承使用过extends实现的

如果是属性，就会直接继承在当前的实例上。方法会保留在类的原型上。

如果子类写了constructor，那么就必须要调用super()方法，在super调用前子类是没有this的。否则会报错，如果不写constructor那么就会隐式地调用。

子类调用super会触发父类的constructor并将参数传递过去

async和await

async可以让我们以一种更加简单的方式写出基于Promise的异步行为,这样就可以不用写promise或者.then这类函数

async可以直接返回一个promise对象，状态是fufiled

特性

  1. async函数内部会返回一个promise对象，如果不是promise，那么它会隐式地包装在promise中（Promise.resolve(‘不是promise的值’)）

  2. await能获取到promise状态改变后地值，如果后面不是一个promise，那么await会把该值转换为已正常处理的（PromisePromise.resolve(‘不是promise的值’).then(()=>undefined）

  3. await后面的promise状态是reject，那么await后的代码不会被执行，async函数将返回状态为reject的promise

  4. async函数内部如果存在await，那么await表达式会暂停整个async函数的执行，等当前promise状态改变后才能恢复

  5. await后面的内容视为异步任务，放到异步调用的宏任务里面，可以看作是传递给settimeout的一个回调函数，等到同步任务执行完了以后再执行。
